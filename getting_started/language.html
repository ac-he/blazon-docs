Statements

Every statement takes the form of the blazon for one coat of arms. Different types of statement are created by different divisions of the field, and the values and parameters of the statement correspond to each of the divisions present.

To the right is an example statement. The division of Per fess; at center an escutcheon represents an arithmetic operation involving two variables. In this case, the variables are represented by the two crowns in chief and the three crowns in base, as detailed in the next section. The operation (in this case, division) is controlled by the color of the escutcheon.

Values

All statement blazons contain one or more values. These are described in terms of their tincture and their charges. Depending on the division in question, the value will be interpreted different ways. Some divisions, like the escutcheon in the example above, use tincture to specify which operation to use. Others, like the base and chief of the example above, use the type of charge and the number of charges to represent a particular number.

Finding the Value

Thus, the values of these three fields are 10, 216, and 1000 respectively.

Variable, Branch Label, or Number?

In the Blazon Programming Language, all variable and branch names are integers in disguise.

The values shown above, if used as variables, could be thought of as Variable10, Variable216, and Variable1000. In reality, a field without divisions represets a branch label, so these would represent the starts of Branch10, Branch216, and Branch1000.

It is important to note that the same value can be reached in a number of ways. For instance, there are five ways to describe the integer 9.

These are all interchangable and will be treated the same by the interpreter. For example, if you assign one value to a variable represented by three mullets, assigning another value to a variable represented by an octofoil will overwrite the variable represented by three mullets. Consider making your code more "readable" by sticking to one representation, or vary it for aesthetic reasons!

Whether a number is interpreted as a variable, branch label or numerical value depends on the division of the field. Output, Arithmetic, and Branching operations all have two possible divisions of the field: one for variables, one for numerical values.

The division per bend prints a variable. The statement shown to the right would print the value stored in Variable99 as a character.

The division per bend sinister prints a value. The statement shown to the right would print the value 99 as a character, which is 'c'.

Syntax

General Patterns

Every division of field will have different syntax. Generally, however, the descriptions of fields in blazon proceeds from top to bottom and from left to right. And no matter what, the blazon will always begin with a capital letter -- the only one in the blazon -- and end with a period.

Often, field tinctures are specified before all divisions and charges, in a list. The blazon of the example to the left begins Per pall argent, sable, and azure; with descriptions of each charge following the semicolon.

When the field tinctures are specified ahead of time, then descriptions of each division of the field follow. In this example these are in chief an anchor, to dexter a scallop, to sinister two ships. The exact wording of each changes for different divisions of the field so it is important to consult the documentation to make sure you get it right!

Charges

Finally, charges are required of most divisions and supported on all of them. These will generally, but not always, follow a similar pattern. Examples of how to use each charge, in plural and singular forms, are given in the big list of charges.

In general, all charges will follow the same pattern:
⮚ The field tincture if not specified ahead of time, followed by a comma.
⮚ The article "a" or a number of charges "one", "two", or "three"
⮚ The name of the charge, pluralized if there are more than one present.
